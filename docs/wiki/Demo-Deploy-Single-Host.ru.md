# Demo Deploy (Single Host) — Пошаговый гид (RU)

Это маршрут "скачать → запустить → торговать".
Никаких командных строк вручную — только двойной клик по файлу `.bat`.

## Структура: два слоя на одном компьютере

Система запускается в два этапа:

1. **Сборка №1 — центр сигналов** — должна стартовать **первой**.
2. **Сборка №2 — торговая группа** — основной интерфейс и база данных.

После запуска обоих слоёв интерфейс открывается в обычном браузере.

---

## 0) Что нужно заранее

### Docker Desktop

Скачайте и установите [Docker Desktop](https://www.docker.com/products/docker-desktop/).

После установки значок Docker появится в системном трее (правый нижний угол экрана).
Подождите, пока значок перестанет «крутиться» — это означает, что Docker готов.

### Архивы проекта

1. Создайте папку `C:\Trading`.
2. Скачайте ZIP-архив `trading-platform-php` и распакуйте в `C:\Trading`.
3. Скачайте ZIP-архив `alpet-libs-php` и распакуйте в `C:\Trading`.

После распаковки структура будет примерно такой:

```
C:\Trading\
    trading-platform-php-main\   ← или trading-platform-php
    alpet-libs-php-main\         ← или alpet-libs-php
```

> **Примечание:** GitHub обычно добавляет `-main` или `-master` к имени папки при разархивации.
> Скрипты находят нужные папки автоматически — переименовывать не нужно.

## ⚠️ Типичные ловушки — прочитайте перед запуском

| # | Ситуация | Что делать |
|---|----------|------------|
| 1 | Двойной клик по `.ps1` открывает Блокнот или ничего не происходит | Используйте `.bat` файлы из папки `scripts\` — они специально созданы для запуска из Проводника |
| 2 | Docker Desktop не запущен — скрипт сразу падает с ошибкой | Откройте Docker Desktop из меню «Пуск», дождитесь значка в трее |
| 3 | Окно с прогрессом мигнуло и закрылось | Нет прав или ошибка в пути — `.bat` держит окно открытым до нажатия Enter |
| 4 | Windows Firewall спрашивает «Разрешить доступ?» | Нажмите **Разрешить** — контейнерам нужен доступ к портам |
| 5 | Страница в браузере не открывается сразу после скрипта | Подождите 1–2 минуты и нажмите F5 — базы данных ещё инициализируются |

## 1) Шаг A: Центр сигналов (Сборка №1)

Откройте папку `scripts\` в Проводнике Windows и **дважды кликните**:

```
start-signals.bat
```

Откроется окно с прогрессом. Ждите строку `#SUCCESS` в конце.

*Linux / Git Bash:*
```bash
sh scripts/deploy-signals-legacy.sh
```

---

## 2) Шаг B: Торговая группа (Сборка №2)

Только после успешного завершения Шага A дважды кликните:

```
start-trading.bat
```

Скрипт автоматически:
- Создаст `.env` с безопасными паролями
- Соберёт и запустит контейнеры базы данных и веб-интерфейса
- Выполнит проверку доступности

Дождитесь строки `#SUCCESS`.

*Linux / Git Bash:*
```bash
sh scripts/deploy-simple.sh
```

---

## 3) Открываем в браузере

После завершения обоих скриптов откройте браузер и проверьте:

| Что | Адрес |
|-----|-------|
| Интерфейс управления | http://localhost:8088/basic-admin.php |
| API | http://localhost:8088/api/index.php |
| Центр сигналов | http://localhost:8090/docs.html |

Если страница не открывается — подождите 1–2 минуты и обновите браузер (F5).

---

## 4) Если что-то не работает

Откройте Command Prompt (`Win+R` → `cmd` → Enter) и выполните:

```
docker compose ps
```

Все контейнеры должны показывать статус `running`.
Если какой-то показывает `Exited` — посмотрите на его логи:

```
docker compose logs --tail 50 web
docker compose logs --tail 50 signals-legacy
```
